An **RPC endpoint** allows developers to interact with a blockchain network by sending requests and receiving data. Endpoints can be either **archival** (storing all historical data) or **non-archival** (storing only recent data for efficiency). 

WebSocket (`wss://`) endpoints enable real-time event streaming, while HTTP (`https://`) endpoints are used for standard requests.

Below is a list of available Gateway.fm RPC endpoints across different regions.

## **Europe (Zurich)**

### **Gnosis Mainnet** (Non-Archival)
- **HTTP:** `https://rpc.eu-central-2.gateway.fm/v4/gnosis/non-archival/mainnet`
- **WebSocket:** `wss://rpc.eu-central-2.gateway.fm/ws/v4/gnosis/non-archival/mainnet`

### **Gnosis Chiado Testnet** (Archival)
- **HTTP:** `https://rpc.eu-central-2.gateway.fm/v4/gnosis/archival/chiado`
- **WebSocket:** `wss://rpc.eu-central-2.gateway.fm/ws/v4/gnosis/archival/chiado`

### **Palm Mainnet** (Archival)
- **HTTP:** `https://rpc.eu-central-2.gateway.fm/v4/palm/archival/mainnet`
- **WebSocket:** `wss://rpc.eu-central-2.gateway.fm/ws/v4/palm/archival/mainnet`

### **Palm Testnet** (Archival)
- **HTTP:** `https://rpc.eu-central-2.gateway.fm/v4/palm/archival/testnet`
- **WebSocket:** `wss://rpc.eu-central-2.gateway.fm/ws/v4/palm/archival/testnet`
